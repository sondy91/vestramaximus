# Lessons learned from vibe coding

## What went well

- Spec file I think gave a good structure to implementing features in a decent order.
- Small atomic commits help to revert or incremently build

## What didn't go well

- Should of started with the data structures and kept a smaller scope
- Don't build out a lot of backend implementations up front, do front and backend for a feature to see if that piece works

## Action Items

- Rethink the data structures for the backend as the user experience isn't the best
- Maybe start front and go down the stack
